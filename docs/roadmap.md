# WebDeck 로드맵 — PowerPoint 수준을 향한 기능 확장

2026-07-03 작성. MVP + Plan 4(새 문서·템플릿 시작·다른 이름으로 저장) 완료 시점 기준.

## 0. 전제와 판단 기준

**목표는 "PowerPoint 복제"가 아니라, 회사 보고 시나리오에서 PPT 없이 문서를 만들고 → 다듬고 → 보고할 수 있는 수준.** AI가 초안을 생성한다는 전제가 있으므로, PPT의 모든 기능이 아니라 "AI가 만든 것을 사람이 고칠 때 아쉬운 것"부터 채운다.

유지해야 할 제약:

- 표준 포맷 v1(self-contained HTML) + 왕복 계약(모델 동등성, 검증기 통과) — 모든 기능 확장은 이 계약을 깨지 않아야 함
- 에디터 런타임 의존성은 react/react-dom 2개
- 사용자는 비개발자 — 기능이 늘어도 툴바는 PPT 문법을 따를 것

각 항목 표기: **중요도**(PPT 체감 기여 상/중/하) / **난이도**(S 반나절급, M 태스크 여러 개, L 플랜 하나급, XL 플랜 여러 개) / **포맷 확장**(v1 스펙·검증기·직렬화 변경 필요 여부)

### D. 표 — PPT 핵심이지만 전략적 판단 필요

보고 문서에 표는 필수다. 다만 표 편집기(행/열 추가·삭제, 셀 병합, 셀 서식)는 그 자체로 플랜 하나급(L)이다. 현재도 AI가 생성한 `<table>`은 opaque 요소로 **보존·이동·크기 조절이 되므로**, 단기에는 "표 내용 수정은 AI에게 요청"(→ I. AI 연동)으로 우회하고, 실사용에서 표 직접 편집 요구가 확인되면 `el-table` 타입 신설(v1.1)로 착수하는 것을 제안.

→ 2026-07-05 Plan 9b로 `el-table` 편집기 구현 완료 (병합 포함, opaque 표 변환 버튼 제공).

### E. 차트 — 자체 편집기는 만들지 않는 방향 제안

사업 보고의 차트 수요는 크지만, 차트 편집기는 XL급이고 PPT조차 Excel에 위임하는 영역. WebDeck의 노선은:

1. AI가 SVG 차트를 생성 → opaque 보존 (현재 동작)
2. 수치 변경·스타일 변경은 AI 수정 요청으로 (→ I)
3. 자체 차트 편집기는 **비목표** (§4)

### G. 발표·보기 — 현재 가장 뚜렷한 공백

| 기능 | 중요도 | 난이도 | 포맷 확장 | 메모 |
|---|---|---|---|---|
| 전체화면 발표 모드(한 장씩, ←→/Esc) | **상** | M | **v1.1(런타임)** | PPT의 F5. 현재는 스크롤 보기뿐이라 "보고 자리에서 띄우기"가 어색함 |
| 슬라이드 전환 효과(없음/fade/push) | 중 | M | v1.1 | CSS 전환 + `data-transition` |
| 요소 애니메이션(클릭당 나타내기) | 중 | L | v1.1 | 회사 보고 문화에선 우선순위 중간 — 전환 효과 후 검토 |
| 발표자 뷰(노트+다음 장+타이머) | 중 | L | — | 별도 창 동기화. 슬라이드 노트(F) 이후 |

**아키텍처 유의**: 뷰어 런타임은 각 문서 안에 인라인으로 박제된다. 런타임을 개선해도 기존 문서는 옛 런타임을 갖고 있으므로, **에디터가 저장할 때 문서의 런타임 스크립트를 최신으로 교체하는 정책**(왕복 계약과의 조화 포함)을 발표 모드 도입 시점에 반드시 함께 결정해야 한다. 이 결정이 G 영역 전체의 선행 과제다.

### H. AI 수정 연동 — PPT에는 없는 차별화 축

`ai/adapter.ts` 인터페이스만 있는 상태. 구현 시나리오: 요소/슬라이드 선택 → 자연어 지시("이 문단 더 간결하게", "이 차트 막대를 파란색으로") → 사내 AI API 호출 → 반환 조각을 파서·왕복 검증 통과 시에만 적용(실패 시 거부) → undo 1스텝.

이 축이 서면 **표·차트 편집기 부재를 상당 부분 상쇄**하므로 (D·E의 우회로), 난이도 L임에도 로드맵 중반에 배치할 가치가 있다.

## 3. 로드맵 — 플랜 단위 제안

실사용 피드백에 따라 순서 조정을 전제로 한 기본 순서. 각 플랜은 기존 워크플로(브레인스토밍 → 플랜 작성 → 서브에이전트 실행 → 최종 리뷰)를 따른다.

### Plan 5 — 서식·속성 심화 ✅ (완료)
폰트 선택, 크기/색 자유 입력, 글머리 목록, 줄 간격, 세로 정렬, 도형 채우기/테두리, 그림자/투명도, 속성 패널(수치 입력), 간격 분배, Shift 비율 리사이즈. 백로그 소형 항목(리사이즈 스냅, 수동 줌) 동반 처리.
— 포맷 확장 없음(전부 extraStyle/기존 모델 범위), 순수 에디터 작업이라 리스크 낮음.

### Plan 6 — 발표 모드와 런타임 갱신 ✅ (완료)
전체화면 발표 모드, 전환 효과, **저장 시 런타임 최신화 정책**(선행 설계 결정), 슬라이드 노트. 포맷 v1.1 첫 확장 — 스펙·검증기·왕복 계약 갱신 규율을 이 플랜에서 확립.

— 2026-07-03 완료. 발표자 뷰·요소 애니메이션은 미착수(로드맵 G 잔여), 슬라이드 노트는 data-notes로 포함됨.

### Plan 7 — 테마·레이아웃·템플릿 ✅ (완료)
테마 색/폰트 편집 UI, 슬라이드 레이아웃 선택, 배경 이미지, 커스텀 템플릿 등록, 시작 화면 썸네일. "회사 표준 양식으로 빠르게"를 완성.

— 2026-07-04 완료. 슬라이드 크기 변경(4:3/A4)은 유예 유지, 슬라이드 노트는 Plan 6에서 data-notes로 선반영됨.

### Plan 8 — AI 수정 연동 (차별화)
adapter 구현 + 선택→지시→미리보기→적용 UI, opaque 요소(표·차트)의 AI 경유 수정. 사내 AI API 확정이 선행 조건.
— **2026-07-04 결정: 맨 마지막으로 이월** (진행 순서: Plan 9 → Plan 10 → Plan 8). 사내 AI API 일정 불확실성을 고려해 나머지를 먼저 완성한다.

### Plan 9 — 큰 요소 타입 ✅ (완료 — 9: 도형·회전, 9b: 표 편집기)
도형 종류 확장(타원·둥근 사각형·선/화살표), 회전, (요구 확인 시) 표 편집기 `el-table`. 포맷 v1.1 요소 확장의 본대.

— 2026-07-04 도형 5종·회전 완료, 2026-07-05 표 편집기(el-table, 병합 포함) 완료. 회전 요소의 드래그 리사이즈는 제한(패널 수치로).

### Plan 9c — 선 서식·드래그 그리기 ✅ (완료)
선 굵기·파선/점선, 화살표 머리 시작/끝 토글, 드래그 그리기 모드(두 점 → frame+rotation 환산). Plan 9가 "후속"으로 미룬 선 고도화 — 2026-07-07 사용자 피드백("선이 단조롭다")으로 착수·완료. 스펙: `superpowers/specs/2026-07-07-webdeck-line-styling-design.md`

— 2026-07-07 완료. 포맷은 line/arrow의 선택 data 속성 4종(기본값 생략 — 기존 문서 무변경), 끝점 핸들 편집은 백로그.

### Plan 9d — 꺾인 연결선·곡선 (요구 확인 시)
ㄱ자 연결선(elbow)·곡선 — 직선의 frame+rotation 모델로는 안 되고 경유점(waypoint) 모델 신설이 필요한 별개 규모라 Plan 9c에서 분리. 도형 사이를 잇는 다이어그램 요구가 실사용에서 확인되면 착수. 도형에 붙어 따라다니는 "연결" 기능 포함 여부는 그때 결정.

### Plan 10 — 통합·배포
사내망 호스팅(+XSS 재평가), (요구 확인 시) PPTX 내보내기, 이미지 내보내기.

**순서 조정 기준**: ① 보고 자리에서 띄울 일이 먼저 생기면 Plan 6을 5보다 앞으로. ② 사내 AI API가 먼저 준비되면 Plan 8을 앞으로 — AI 수정이 D·E 우회로라서 조기 투입 가치가 있음. ③ "제출은 pptx" 요구가 확인되면 PPTX 내보내기를 독립 플랜으로 승격.

## 4. 비목표 — 하지 않기로 제안하는 것

- **실시간 공동 편집** — CRDT/서버 인프라(XL). 파일 기반·self-contained라는 정체성과 충돌. 공유는 파일로.
- **PPTX 가져오기** — 완전 변환 불가능한 늪(XL). 필요하면 AI에게 변환을 맡기는 경로로.
- **자체 차트 편집기** — PPT도 Excel에 위임하는 영역. AI 생성+AI 수정으로 대체.
- **완전한 애니메이션 엔진**(경로 이동, 트리거 체인 등) — 전환 효과+나타내기 수준에서 멈춘다.
- **매크로/스크립팅** — 범위 밖.
