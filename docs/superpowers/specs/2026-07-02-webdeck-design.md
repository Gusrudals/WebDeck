# WebDeck 설계 문서

- 작성일: 2026-07-02
- 상태: 승인됨 (사용자 승인 완료)

## 1. 배경과 문제 정의

AI Agent의 문서 산출물은 대부분 마크다운인데, 마크다운은 레이아웃 제어·디자인 표현력·발표 단위 구성에서 한계가 있다. 한국 기업의 보고 문화는 PPT 중심이지만, 현재 AI의 PPT 생성 품질(특히 이미지 기반 PPT)은 편집이 불가능해 기업 요구를 만족하지 못한다. HTML은 LLM이 가장 잘 생성하는 시각 포맷이지만, **비개발자가 HTML을 직접 고칠 수 없다**는 결정적 단점이 있다.

**목표: HTML을 PPT로 변환하는 것이 아니라, HTML 문서를 그대로 PPT처럼 활용한다.** AI가 표준 포맷의 HTML 슬라이드 문서를 생성하고, 비개발자가 PowerPoint와 최대한 비슷한 WYSIWYG 편집기로 그 HTML을 직접 편집한다.

## 2. 요구사항

- **목표 수준**: 회사에서 실제 사용할 도구 (보안·비개발자 사용성 중요)
- **주 사용자**: 비개발 현업 직원 전반 — 마우스 중심의 완전한 WYSIWYG 필수
- **필수 편집 기능**: 텍스트 수정, 요소 이동/크기 조절, 슬라이드 단위 관리(추가/삭제/복제/순서), AI 수정 요청(구조만 준비, 구현 보류)
- **편집 경험**: Microsoft PowerPoint와 최대한 비슷하게
- **배포**: 일단 로컬 실행. 서버 의존을 최소화해 추후 어디로든(사내망 정적 호스팅 등) 이전 가능하게 설계
- **AI 백엔드**: MVP에서는 보류. 확장 지점만 마련

## 3. 검토한 대안과 결정

| 대안 | 내용 | 결정 |
|------|------|------|
| A안: 자기완결형 | 문서마다 에디터 런타임 내장 | 기각 — 저장 UX 열악, PPT급 UI를 파일에 내장하기 어려움, 에디터 버전 파편화 |
| **B안: 에디터/문서 분리** | 정적 웹 에디터 앱 + 표준 문서 포맷 | **채택** — PPT급 UX 구현 가능, 문서는 가볍게 유지, 에디터 독립 진화 |
| C안: B + 문서에 초경량 뷰어 내장 | 발표 모드·인쇄·"편집하기" 버튼 내장 | 추후 확장 — 포맷에 뷰어 CSS/JS를 처음부터 포함해 확장 씨앗을 심음 |

입력 포맷은 **임의 HTML 수용이 아닌 표준 포맷 기반**으로 한다. 임의 HTML은 구조가 제각각이라 요소 이동/크기 조절 같은 WYSIWYG 편집이 사실상 불가능하다. 대신 AI Agent용 생성 가이드를 제공해 AI가 표준 포맷으로 생성하게 한다. 임의 HTML 임포트(AI 변환)는 추후 기능.

## 4. 전체 아키텍처

```
┌─────────────────┐     생성      ┌──────────────────────┐
│  AI Agent        │ ──────────▶ │  문서 (.html)          │ ◀── 브라우저만으로 보기/인쇄
│  + 생성 가이드     │             │  WebDeck 표준 포맷      │
└─────────────────┘             └──────────┬───────────┘
                                          │ 열기/저장
                                ┌──────────▼───────────┐
                                │  에디터 앱 (정적 SPA)    │
                                │  PPT급 WYSIWYG 편집     │
                                └──────────────────────┘
```

세 가지 산출물:

1. **WebDeck 문서 포맷 (v1)** — 자기완결형 단일 .html. 에디터 없이 아무 브라우저에서 온전히 보임
2. **에디터 앱** — 서버 없는 순수 정적 SPA. 파일 I/O는 File System Access API
3. **AI 생성 가이드** — 포맷 명세 + 예제 템플릿 + 검증 스크립트

서버가 없으므로 보안 검토 부담이 최소이고, 로컬 실행 → 사내망 정적 호스팅 이전이 빌드 산출물 복사만으로 가능하다.

## 5. 문서 포맷 (WebDeck HTML v1)

PPT의 문서 모델을 HTML로 1:1 대응시킨다.

### 5.1 구조

```html
<!DOCTYPE html>
<html lang="ko" data-webdeck-version="1">
<head>
  <meta charset="utf-8">
  <title>문서 제목</title>
  <style>/* 테마 변수 + 뷰어 CSS (인라인) */</style>
</head>
<body>
  <main class="deck" data-slide-width="1280" data-slide-height="720">
    <section class="slide" data-bg="#ffffff">
      <div class="el el-text" style="left:96px; top:80px; width:1088px; height:120px;">
        <p><strong>2026년 상반기 실적 보고</strong></p>
      </div>
      <div class="el el-image" style="left:96px; top:240px; width:520px; height:360px;">
        <img src="data:image/png;base64,..." alt="실적 차트">
      </div>
      <div class="el el-shape" data-shape="rect"
           style="left:660px; top:240px; width:524px; height:360px; background:var(--wd-accent);"></div>
    </section>
    <!-- 슬라이드 반복 -->
  </main>
  <script>/* 초경량 뷰어 스크립트 (인라인, 수십 줄) */</script>
</body>
</html>
```

### 5.2 규칙

- **캔버스**: 슬라이드당 고정 논리 캔버스 1280×720px (16:9). 포맷 차원에서는 `data-slide-width/height`로 크기를 명시해 추후 4:3 등 지원 여지를 두되, MVP 에디터는 16:9만 지원
- **요소 = 절대 위치 박스**: 모든 요소는 `left/top/width/height` 인라인 스타일을 가진 `.el` div. z-order는 DOM 순서로 표현. PPT의 요소 모델과 1:1 대응
- **요소 타입 (v1)**: 텍스트 상자(`el-text`), 이미지(`el-image`), 도형(`el-shape`, v1은 사각형만) + 슬라이드 배경색(`data-bg`)
- **텍스트 서식 (v1)**: 굵게/기울임/밑줄, 글자 크기, 글자 색, 정렬(좌/중/우) — `<strong>`, `<em>`, `<u>`, span 인라인 스타일로 표현
- **테마 = CSS 변수**: 색상·폰트를 `:root`의 `--wd-*` 변수로 중앙화 (`--wd-primary`, `--wd-accent`, `--wd-text`, `--wd-font-body`, `--wd-font-heading` 등). 브랜드 테마 적용이 한 곳 수정으로 가능
- **이미지는 data URI 내장**: PPT처럼 단일 파일 유통 보장
- **뷰어 내장**: 인라인 CSS로 슬라이드를 세로 나열, 인라인 JS(수십 줄)로 뷰포트 폭에 맞춰 transform scale. print CSS로 슬라이드 1장 = PDF 1페이지 (브라우저 인쇄 → PDF 내보내기)
- **버전 명시**: `data-webdeck-version="1"` — 포맷 진화 시 하위 호환 처리 근거

## 6. 에디터 앱

### 6.1 스택과 실행

- React + TypeScript + Vite
- 개발: `npm run dev`. 배포: `npm run build` 산출물(정적 파일)을 아무 정적 호스팅에 복사
- 파일 열기/저장: File System Access API (Chrome/Edge 기준 — 한국 기업 표준 브라우저). 미지원 브라우저는 업로드/다운로드 폴백

### 6.2 핵심 아키텍처 — 모델이 진실, HTML은 저장 포맷

```
.html 파일 ──파서──▶ 내부 JSON 모델 (deck → slides → elements)
                        │  편집은 모델에서만 발생 (React 렌더)
.html 파일 ◀─직렬화── 모델
```

- DOM을 직접 편집하지 않는다. undo/redo·안정성 확보의 핵심
- undo/redo: 불변(immutable) 모델 스냅샷 스택
- **왕복 보존 원칙**: 파서가 이해하지 못하는 요소/속성은 삭제하지 않고 불투명(opaque) 블록으로 모델에 보존, 직렬화 시 원문 그대로 출력. 에디터가 문서를 파괴하는 일이 없도록 한다

### 6.3 편집 기능 (PPT 대응)

| 영역 | 기능 |
|------|------|
| 캔버스 | 요소 선택(클릭), 다중 선택(Shift), 드래그 이동, 8방향 리사이즈 핸들, 스냅 가이드(슬라이드 중앙/가장자리/타 요소 정렬), 줌 |
| 텍스트 | 더블클릭 → 제자리(in-place) 편집, 굵게/기울임/밑줄/크기/색/정렬 |
| 슬라이드 패널(좌측) | 썸네일 목록, 추가/삭제/복제, 드래그로 순서 변경 |
| 툴바(상단) | 텍스트 서식, 삽입(텍스트 상자/이미지/도형), 정렬 도구, 앞으로/뒤로 보내기 |
| 단축키 | Ctrl+Z/Y(undo/redo), Ctrl+C/V/D(복사/붙여넣기/복제), Delete, 방향키 미세 이동 — PPT와 동일 |

### 6.4 AI 확장 지점

`AIServiceAdapter` 인터페이스만 정의하고 MVP에서는 미구현. 추후 사내 LLM 프록시든 Claude API든 어댑터 구현 하나로 "AI에게 수정 요청" 기능을 붙일 수 있는 구조. UI에는 관련 버튼을 노출하지 않는다.

## 7. AI 생성 가이드

- `docs/ai-guide.md`: 포맷 명세, 좌표 체계, 요소 배치 규칙, 테마 변수 사용법, 한국 보고서 단골 레이아웃 예제(표지 / 목차 / 본문 2단 / 요약)
- 템플릿 3종: 실제로 열리는 완성 예제 .html 파일 (`templates/`)
- 검증 스크립트 `tools/validate-webdeck.mjs` (Node): AI가 생성한 파일의 포맷 적합성을 검사해 오류 목록 출력. AI Agent가 생성 → 검증 → 수정 루프를 스스로 돌 수 있게 한다

## 8. 에러 처리

- 포맷이 아닌 파일 열기 → 명확한 오류 메시지와 함께 거부 (크래시 금지)
- 알 수 없는 요소/속성 → 왕복 보존 원칙에 따라 보존 + 사용자에게 "편집 불가 요소 N개 보존됨" 안내
- 저장 실패(권한 등) → 오류 안내 + 다운로드 폴백 제안
- undo 이력은 세션 내 유지 (파일에 저장하지 않음)

## 9. 테스트 전략

- **파서/직렬화 왕복 테스트 (최우선)**: 모델 → HTML → 모델 동일성, 골든 파일(샘플 문서) 기반. 문서 손상 방지의 생명선
- 모델 조작(요소 이동/크기/슬라이드 순서 등) 단위 테스트: Vitest
- 에디터 인터랙션: Vitest + React Testing Library (드래그 계열 e2e는 추후 Playwright)
- 검증 스크립트 자체 테스트: 정상/오류 샘플 파일 기반

## 10. MVP 범위

**포함**
- 문서 포맷 v1: 텍스트 상자, 이미지, 사각형 도형, 슬라이드 배경색, 테마 변수, 내장 뷰어(세로 나열 + 스케일) + 인쇄 PDF
- 에디터: §6.3의 편집 기능 전부, 파일 열기/저장(FSA API + 폴백), undo/redo
- AI 생성 가이드 + 템플릿 3종 + 검증 스크립트
- 파서/직렬화 왕복 테스트

**제외 (추후 단계)**
- AI 수정 요청 (어댑터 인터페이스만)
- 차트/표/선·원 등 추가 도형, 요소 회전, 그룹화
- 발표 모드, 슬라이드 전환 애니메이션
- 임의 HTML 임포트(AI 변환)
- 동시 편집/코멘트, 사내망 배포 자동화

## 11. 리포지토리 구조 (예정)

```
webdeck/
├── docs/
│   ├── ai-guide.md              # AI Agent용 생성 가이드
│   └── superpowers/specs/       # 설계 문서 (본 문서)
├── templates/                   # 완성 예제 문서 3종
├── tools/
│   └── validate-webdeck.mjs     # 포맷 검증 스크립트
└── editor/                      # React + TS + Vite 에디터 앱
    └── src/
        ├── model/               # 문서 모델, 파서, 직렬화
        ├── canvas/              # 캔버스, 선택/드래그/리사이즈
        ├── panels/              # 슬라이드 패널, 툴바
        └── ai/                  # AIServiceAdapter 인터페이스 (미구현)
```
