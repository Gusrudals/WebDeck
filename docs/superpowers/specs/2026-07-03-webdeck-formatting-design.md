# WebDeck Plan 5 설계 — 서식·속성 심화

2026-07-03 브레인스토밍 승인본. 마스터 설계(`2026-07-02-webdeck-design.md`)를 보완하는 확장 스펙이며, 완료 시 마스터 스펙 §12에 이력을 추가한다.

## 1. 목표와 범위

"PPT 같다"는 체감을 좌우하는 서식·속성 기능을 채운다. **포맷 v1 확장 없음** — 모든 신규 스타일은 기존 모델(extraStyle/인라인 스타일)과 왕복 계약 범위 안에서 동작한다.

포함:

1. **텍스트 서식 심화** — 폰트 선택, 글자 크기 자유 입력, 글자색 팝오버, 글머리/번호 목록, 줄 간격, 세로 정렬
2. **요소 공통 스타일** — 채우기(배경), 테두리, 그림자, 투명도
3. **속성 패널** — 우측 고정 패널, x/y/w/h 수치 입력 + 스타일 UI
4. **조작 개선** — 간격 균등 분배, Shift 비율 리사이즈, 리사이즈 스냅, 수동 줌

비범위 (백로그 유지):

- 편집 모드 밖에서의 **인라인** 서식(굵게/크기/색) 적용 — 저장된 HTML 변환이 필요해 제외. 블록/박스 수준(세로 정렬, 줄 간격, 공통 스타일)은 요소 선택만으로 적용 가능
- 자체 셀렉션 서식 엔진 — execCommand를 이번 플랜까지 유지
- 웹폰트 임베딩 — self-contained 제약. 시스템 폰트 스택만
- 이미지 자르기·교체, 도형 종류 확장, 회전 (로드맵 Plan 9)

## 2. UI 골격 — 우측 속성 패널 (승인: A안)

레이아웃을 `topbar / toolbar / side(좌: 슬라이드) | canvas | props(우: 속성)` 그리드로 확장한다. 속성 패널은 선택 상태에 따라 세 모드로 렌더링:

| 선택 상태 | 패널 내용 |
|---|---|
| 없음 | 슬라이드 속성 — 배경색(기존 툴바의 배경색 UI를 패널로 이전) |
| 단일 요소 | 위치·크기(x/y/w/h 수치 입력) + 공통 스타일(채우기/테두리/그림자/투명도) |
| 다중 요소 | 공통 스타일만 (일괄 적용) |

- 툴바는 유지하되 역할을 나눈다: 툴바 = 삽입·텍스트 서식·정렬·z순서(자주 쓰는 동작), 패널 = 속성·스타일(선택 대상의 상태 표시와 정밀 조정)
- 패널 컴포넌트: `editor/src/panels/PropertiesPanel.tsx` (신규). 파일이 커지면 섹션별 하위 컴포넌트로 분리

## 3. 속성 패널 상세

### 3.1 위치·크기 수치 입력

- x/y/w/h 네 개의 숫자 입력. 표시는 현재 frame 값(소수 첫째 자리 반올림 표시, 저장은 원 값 유지)
- **커밋 규칙**: blur 또는 Enter에서 1회 APPLY_DOC (배경색 bgDraft 드래프트 패턴 재사용 — 타이핑 중에는 디스패치하지 않음). Escape는 이전 값 복원
- 파싱 실패(빈 값·숫자 아님)는 이전 값 복원, 디스패치 없음
- 클램프: w/h ≥ MIN_SIZE(8). x/y는 제한 없음 — PPT처럼 슬라이드 밖 배치 허용
- 다중 선택 시 위치·크기 섹션은 숨김 (혼합 값 표시의 복잡성 회피)

### 3.2 공통 스타일 — 채우기·테두리·그림자·투명도

PPT와 동일하게 **텍스트 상자·이미지·도형 모두**에 적용한다 (구현은 세 타입 공통: 요소 박스의 extraStyle 갱신).

- **채우기**: 색 팝오버(§4.3과 동일 컴포넌트) + "채우기 없음". `background` 속성
- **테두리**: 두께(0=없음/1/2/4px), 색, 스타일(실선/점선). `border` 속성
- **그림자**: 켜기/끄기 + 프리셋 2종(약하게/강하게). `box-shadow` 속성. 자유 조정은 두지 않는다(YAGNI)
- **투명도**: 슬라이더 0~100%. `opacity` 속성. 드래그 중에는 로컬 상태로 미리보기, pointerup/블러에서 1회 커밋
- 값 표시: 요소의 extraStyle에서 읽어 현재 상태를 보여준다. 파싱 불가한 복잡 값(예: AI가 넣은 그라데이션)은 "사용자 지정"으로 표시하고 덮어쓰기 전까지 보존

## 4. 텍스트 서식 심화

### 4.1 폰트 선택 (툴바 드롭다운)

시스템 폰트 스택 5종 큐레이션. 저장은 스택 문자열 전체(폰트 없는 PC에서 자연 폴백):

| 라벨 | 스택 |
|---|---|
| 기본 고딕 | `"Pretendard", "Malgun Gothic", "맑은 고딕", sans-serif` |
| 나눔고딕 | `"NanumGothic", "나눔고딕", "Malgun Gothic", sans-serif` |
| 명조 | `"NanumMyeongjo", "나눔명조", "Batang", "바탕", serif` |
| 돋움 | `"Dotum", "돋움", "Gulim", "굴림", sans-serif` |
| 고정폭 | `"D2Coding", "Consolas", "Courier New", monospace` |

- 편집 모드 중 선택 영역에 적용: `styleWithCSS` + `fontName` (execColor와 같은 패턴), 인자로 스택 문자열 전달
- 드롭다운은 `keepFocus`(onPointerDown preventDefault) 규칙을 따라 편집 포커스를 유지

### 4.2 글자 크기 자유 입력

- 기존 프리셋 6종 버튼을 **숫자 입력 + 프리셋 드롭다운** 콤보로 교체. 8~120px 클램프
- 적용은 기존 `execFontSize`(fontSize 7 → px span 치환) 재사용

### 4.3 글자색 팝오버

- `input[type=color]`는 포커스를 뺏어 편집이 끝나는 기존 제약 유지 → **자체 팝오버**: 스와치 12종(기존 6 + 보조 6) + hex 텍스트 입력
- 팝오버 전체가 keepFocus 규칙을 따르고, 적용은 기존 `execColor`
- 같은 팝오버 컴포넌트를 공통 스타일 채우기·테두리 색(§3.2)에 재사용 (그쪽은 keepFocus 불필요 — 편집 모드가 아님)

### 4.4 글머리 기호·번호 목록

- execCommand `insertUnorderedList` / `insertOrderedList` 토글 버튼 2개
- 에디터 캔버스 CSS에 `.el-text ul/ol` 여백 규칙 추가. 문서 자체는 브라우저 기본 렌더링으로 뷰어에서도 동일하게 보임 — 포맷 확장 불필요
- 왕복: ul/ol/li는 텍스트 요소 `html` 필드에 그대로 저장 (기존 계약 그대로)

### 4.5 줄 간격

- execCommand에 없으므로 자체 처리: 현재 셀렉션이 걸친 **문단 블록(p, li)** 들을 찾아 `style.lineHeight` 설정
- 프리셋: 1.0 / 1.15 / 1.5 / 2.0 (단위 없는 배수값)
- 편집 모드 전용(셀렉션 기반). 커밋은 기존 텍스트 편집 흐름(blur 시 innerHTML 비교 후 APPLY_DOC)에 자연 포함

### 4.6 세로 정렬 (위/중간/아래)

- 텍스트 상자 박스의 extraStyle에 `display:flex; flex-direction:column; justify-content:flex-start|center|flex-end` 적용
- **요소 선택 상태에서도 적용 가능** (편집 모드 불필요 — 박스 스타일이므로). 툴바의 객체 정렬 옆에 배치
- 인라인 스타일이므로 뷰어에서도 그대로 렌더링. 포맷 확장 불필요

## 5. 조작 개선

### 5.1 간격 균등 분배

- 3개 이상 선택 시 활성. 가로/세로 각 1버튼 (객체 정렬 6종 옆)
- 알고리즘: 해당 축으로 정렬 후 첫/마지막 요소 고정, 사이 간격을 균등화. `geometry.ts`에 순수 함수 `distributeFrames(frames, axis)` 추가
- 1클릭 = 1 APPLY_DOC

### 5.2 Shift 비율 유지 리사이즈

- **모서리 핸들(nw/ne/sw/se) + Shift**에서 시작 frame의 종횡비 고정. 변 핸들은 대상 아님(PPT 동일)
- `resizeFrame`에 `lockAspect` 옵션 추가(순수 함수 확장). 제스처 중 Shift 상태는 pointermove 이벤트의 `shiftKey`로 판정 (누르고 떼는 것 실시간 반영)

### 5.3 리사이즈 스냅

- 단일 요소 리사이즈 시, 움직이는 변을 기존 `buildSnapTargets` 대상(다른 요소 변·슬라이드 경계·중앙)에 흡착. `snapResize(frame, handle, targets)` 순수 함수 추가
- Shift 비율 고정과 동시 적용 시에는 **비율이 우선**(스냅 생략) — 규칙 충돌 회피

### 5.4 수동 줌

- 배율 상태는 CanvasArea 로컬 UI 상태: `'fit' | 0.5 | 0.75 | 1 | 1.5 | 2` (문서 아님 — undo/dirty 무관)
- UI: 캔버스 하단 구석 드롭다운(맞춤/50/75/100/150/200%) + **Ctrl(⌘)+휠 줌**(연속 배율이 아니라 프리셋 단계를 오르내림)
- fit이 아닌 배율에서는 캔버스 영역이 스크롤 컨테이너가 된다. 기존 제스처의 scale 보정 로직은 현재 배율을 그대로 사용(이미 scale 파라미터화되어 있음)

## 6. 상태·데이터 흐름

- 문서 상태 변경은 전부 기존 APPLY_DOC 경로. **신규 리듀서 액션 없음**
- 패널 드래프트 입력(수치·투명도·hex)은 컴포넌트 로컬 상태 → 커밋 시점에만 디스패치 (1 조작 = 1 undo 스텝)
- 줌은 로컬 UI 상태, 클립보드·선택과 무관

## 7. 오류 처리

- 수치 입력: 파싱 실패·범위 밖 → 이전 값 복원(w/h는 클램프), 디스패치 없음
- hex 입력: `#rgb`/`#rrggbb`만 수용, 그 외 무시
- extraStyle의 인식 불가 값: 덮어쓰기 전까지 원문 보존(기존 왕복 원칙), 패널에는 "사용자 지정" 표시
- execCommand 미지원 환경(테스트 등): 기존 옵셔널 체이닝 패턴 유지 — no-op

## 8. 테스트 전략

- **geometry 순수 함수**: `distributeFrames`, `resizeFrame(lockAspect)`, `snapResize` — 단위 테스트 (경계·역전·최소 크기 케이스 포함)
- **속성 패널**: RTL — 수치 입력 blur/Enter 커밋·Escape 복원·클램프, 스타일 버튼 1클릭 1 APPLY_DOC, 다중 선택 모드 전환
- **텍스트 서식**: 기존 Toolbar 테스트 관례(execCommand 스파이) — 폰트/크기/색/목록 호출 검증. 줄 간격은 DOM 조작 결과 검증
- **세로 정렬·공통 스타일**: 요소 extraStyle 갱신 → 직렬화 결과에 반영 확인 (왕복 회귀 테스트 1종 포함)
- **줌**: 배율 변경 시 렌더 scale 반영, 제스처 좌표 보정 회귀 테스트

## 9. 구현 노트

- 색 팝오버는 `editor/src/panels/ColorPopover.tsx`로 분리해 툴바(글자색)와 패널(채우기·테두리)에서 공유
- 툴바가 이미 140줄대 — 서식 섹션 추가 시 `Toolbar.tsx`에서 텍스트 서식 그룹을 하위 컴포넌트로 분리 검토
- 기존 배경색 UI를 툴바에서 패널로 이전할 때, bgDraft 커밋 패턴과 테스트도 함께 이동
