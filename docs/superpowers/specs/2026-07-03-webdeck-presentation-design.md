# WebDeck Plan 6 설계 — 발표 모드와 런타임 갱신

2026-07-03 브레인스토밍 승인본. 마스터 설계(`2026-07-02-webdeck-design.md`)를 보완하는 확장 스펙이며, 완료 시 마스터 스펙 §12에 이력을 추가한다. 포맷 확장 라벨은 **v1.1**이다 (버전 속성은 `"1"` 유지 — §4).

## 1. 범위와 승인된 방향

포함:

1. **문서 자체 전체화면 발표 모드** — 저장된 .html을 브라우저로 열어 발표 (에디터 내 발표 미리보기 없음)
2. **슬라이드 전환 효과** — 없음/페이드/밀기, 슬라이드 단위 `data-transition`
3. **저장 시 런타임 자동 최신화** — 항상 자동 (열기 시 정규화, 저장 시 기록)
4. **슬라이드 노트** — `data-notes` 평문 속성, 속성 패널에서 편집
5. **포맷 v1.1 확장 규율 확립** — 스펙 → 검증기 → 파서/직렬화(왕복) → 뷰어 런타임 → 에디터 UI 5종 세트를 한 플랜에서 함께 갱신

비범위: 발표자 뷰(노트+다음 장+타이머), 요소 애니메이션(나타내기), 에디터 내 발표 미리보기, F5 진입(브라우저 새로고침 충돌 — 단축키는 `P`).

## 2. 런타임 아키텍처

### 2.1 단일 마킹 스크립트

- 런타임 전체를 **`<script data-webdeck-runtime="2">` 단일 스크립트**로 통합한다. v1 런타임 기능(배경 적용·화면 맞춤 줌·인쇄 핸들러)과 발표 모드가 전부 이 안에 있고, **발표용 CSS·발표 버튼도 이 스크립트가 브라우저에서 주입**한다 (DOMParser는 스크립트를 실행하지 않으므로 에디터·문서 모드·저장물은 오염되지 않음)
- head `<style>`은 순수하게 문서 소유(테마 변수·레이아웃·인쇄 CSS)로 남는다 — 교체 지점은 스크립트 하나
- 마커 값은 런타임 버전 정수. 이번 플랜에서 `2` (마킹 없는 기존 스크립트가 사실상 v1)

### 2.2 단일 소스와 동기화

- 에디터에 `editor/src/model/runtime.ts` 신설: `RUNTIME_VERSION`(정수)과 최신 런타임 `<script>` 태그 전체 문자열, 그리고 정규화 함수를 export
- 템플릿 3종(`templates/*.html`)과 `docs/ai-guide.md`의 런타임 블록을 새 런타임으로 교체
- **동기화 강제**: templates.test.ts에 "각 템플릿의 마킹된 런타임 스크립트 == runtime.ts의 최신본" 동등성 테스트 추가 — 한쪽만 고치면 테스트가 깨진다

### 2.3 최신화 정책 (승인: 항상 자동)

- **시점: 문서를 열 때** — `parseWebdeck` 직후 `normalizeRuntime(doc)`으로 bodyScript를 정규화. 이후 왕복 검증·저장·dirty 판정이 전부 정규화된 문서 기준으로 일관되고, 저장하면 최신 런타임이 파일에 기록된다. 열어서 저장하지 않으면 파일 불변
- 정규화 규칙 (bodyScript의 스크립트들을 순서대로 검사):
  1. `data-webdeck-runtime` 마킹 스크립트 → 최신본으로 교체 (버전 같아도 본문 통일)
  2. 마킹은 없지만 **v1 시그니처**(`dataset.slideWidth`와 `beforeprint`를 모두 포함) → 최신본으로 교체
  3. 그 외 스크립트 → 원문 보존 (opaque 보존 원칙)
  4. 1·2에 해당하는 스크립트가 하나도 없고 **bodyScript가 비어 있으면** → 최신본 추가 (런타임 없는 AI 문서 구제). 스크립트가 있는데 인식 실패면 추가하지 않는다 (미인식 커스텀 런타임과의 이중 실행 방지)
  5. 교체 대상이 여럿이면 첫 번째만 최신본으로 바꾸고 나머지 교체 대상은 제거 (중복 런타임 정리)
- 정규화는 멱등: `normalizeRuntime(normalizeRuntime(d)) == normalizeRuntime(d)`
- 새 문서(템플릿 시작)도 동일 경로 통과 — 템플릿이 이미 최신이므로 no-op

## 3. 발표 모드 (런타임 v2 동작)

- **진입**: 문서 우상단 "발표" 버튼(기본 반투명, 호버 시 불투명, 인쇄·발표 중 숨김) 클릭 또는 단축키 `P` (input/textarea 포커스 중엔 무시). 시작 슬라이드는 항상 첫 장
- **조작**: `→`/`Space`/`PageDown` 다음, `←`/`PageUp` 이전, `Home`/`End` 처음/끝, 클릭 다음 장, `Esc` 종료
- **렌더링**: 검은 배경 고정 오버레이에 현재 슬라이드 복제본을 화면에 맞게 scale(중앙 정렬). `requestFullscreen` 시도, 거부(iframe·권한)되면 고정 오버레이만으로 동작. 전체화면 해제 시 발표도 종료
- **전환**: 들어오는 슬라이드의 `data-transition` — `fade`(투명도), `push`(진행 방향에 맞춘 가로 밀기), 없음(즉시). 미지원 값은 없음으로 취급
- 발표 중 원본 deck DOM은 변경하지 않는다 (오버레이는 복제본, 종료 시 오버레이 제거)

## 4. 포맷 v1.1 확장 (하위 호환)

- `data-webdeck-version="1"` **유지**. 신규 속성은 전부 선택적이고 구 런타임에서 무해(렌더링 불변). "v1.1"은 스펙 라벨
- `<section class="slide">` 선택 속성 2종:
  - `data-transition="fade" | "push"` — 없으면 전환 없음
  - `data-notes="평문"` — 슬라이드 노트 (서식 없음, 속성 이스케이프는 기존 직렬화 규칙)
- **모델**: `Slide`에 1급 필드 `transition: 'fade' | 'push' | null`, `notes: string`(없으면 `''`) 추가. 파서는 data-bg처럼 추출(extraAttrs에 남기지 않음), 직렬화는 값이 있을 때만 속성 출력(notes는 비어 있지 않을 때만)
- **미지원 transition 값**: 파서는 1급 필드로 승격하지 않고 extraAttrs에 원문 보존(왕복 무손실), 런타임은 없음으로 취급. 검증기는 오류로 보고
- **검증기**(tools/lib/validate.mjs): `data-transition` 값 검증(fade/push 외 오류), `data-notes` 허용, `script[data-webdeck-runtime]` 허용(기존 인라인 스크립트 허용 규칙 그대로)

## 5. 에디터 UI

- 속성 패널의 **슬라이드 모드**(선택 없음)에 추가:
  - **전환 효과** select (없음/페이드/밀기) — 1클릭 = 1 APPLY_DOC, 같은 값 재선택은 no-dispatch
  - **노트** textarea — 드래프트 패턴(입력 중 디스패치 없음, blur 커밋, Escape 취소, 같은 값 커밋은 no-dispatch). 슬라이드 전환 시 드래프트 초기화(슬라이드별 key)
- ops 신설(순수 함수, 기존 mapSlide 패턴): `setSlideTransition(doc, slideId, transition)`, `setSlideNotes(doc, slideId, notes)`

## 6. 오류 처리·알려진 한계

- 전체화면 요청 거부 → 고정 오버레이 폴백 (기능 동일, 브라우저 UI만 남음)
- 발표 모드는 저장된 문서에서만 — 에디터에서 확인하려면 저장 후 브라우저로 열기
- 런타임 정규화 인식 실패 문서(커스텀 스크립트만 존재)는 발표 모드가 생기지 않음 — 보존 우선
- 노트는 발표 중 표시되지 않음 (발표자 뷰가 후속)
- zoom 기반 화면 맞춤(v1 방식)은 유지하되 발표 오버레이는 transform scale 사용 — 문서 스크롤 보기와 발표 보기의 렌더링 경로는 독립

## 7. 테스트 전략

- **파서/직렬화**: transition·notes 왕복, extraAttrs 미오염, 미지원 transition 값의 extraAttrs 보존, 구버전 문서(속성 없음) 회귀, checkRoundTrip 통과
- **normalizeRuntime**: 마킹 구버전 교체, v1 시그니처 교체, 커스텀 스크립트 보존(순서 포함), 빈 bodyScript 추가, 미인식+비어있지 않음 → 무변경, 중복 런타임 정리, 멱등성
- **동기화**: 템플릿 3종의 마킹 스크립트 == runtime.ts 최신본, 템플릿 validate 통과(tools 테스트)
- **검증기**: data-transition 값 검증, data-notes·마킹 스크립트 허용
- **에디터 UI**: 전환 select·노트 드래프트의 RTL 테스트 (1 APPLY_DOC, no-dispatch 가드, Escape)
- **런타임 스크립트 자체**: 브라우저 전용 동작(fullscreen·keydown·전환)은 단위 테스트 불가 — 수동 확인 목록에 명시. 구조 보증은 템플릿 validate + 동기화 테스트로
